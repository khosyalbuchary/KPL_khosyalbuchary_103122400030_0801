# Tugas
membuat fungsi yang menolak bilangan-bilangan kelipatan 3, 5, atau 15, menerima bilangan-bilangan bukan "fizz buzz", dan melempar yang bukan bilangan bulat.

# Deskripsi
Program ini mengimplementasikan fungsi is_not_fizzbuzz dengan menerapkan prinsip Design by Contract (DbC) dan Defensive Programming untuk memastikan validitas data sebelum diproses. Melalui penerapan prakondisi yang ketat, program akan memvalidasi bahwa input harus berupa bilangan bulat dan akan melemparkan TypeError jika aturan tersebut dilanggar, sementara logika intinya bertugas menentukan apakah sebuah angka tidak habis dibagi tiga atau lima. Seluruh proses ini dibungkus dalam mekanisme exception handling menggunakan blok try-catch guna menjamin aplikasi tetap berjalan stabil dan mampu menangani kesalahan input secara informatif tanpa menyebabkan kegagalan sistem secara keseluruhan.
